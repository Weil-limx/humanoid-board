01 / 概览

绕过去，还是跨过去

本题考察人形机器人面对特殊障碍物（箱子）时的全身运控能力。在传统人形 locomotion benchmark 基础上引入两点： 基于避障的灵活绕行（路径重规划完全避开箱子、绕至目标区），与 运动模态的混合挑战（允许跨上箱子再跳下——箱沿有意识降重心，跳下时双脚同时落地缓解冲击）。

箱子固定（横跨赛道置于正中）。机器人既可绕行（到达终点即得主分），也可跨越（额外拿跨越奖励）—— 能否读准箱体、稳稳跨上又落得干净，是高分与否的关键。

02 / 赛题

赛道与机器人

约 20m 长、8m 宽的平坦直线赛道，正中横跨一个固定箱子。机器人可任意方式通过：绕行、跨越、借助手臂撑扶均合法。

赛道平坦 20 m × 8 m 直线，中线 y = 0。机器人 spawn 于 x = −9，终点线 x = +9。
箱子进深 0.74 m × 宽 2.0 m × 高 0.38 m，横跨赛道置于正中（x = 0），固定（不可移动）。

机器人与约束

本体Oli EDU 人形机器人。
传感器自带胸口 + 头部 Intel RealSense D435i 深度相机。
手臂使用合法：可推、可撑、可抱（手臂与双脚为合法接触部位，不计罚）。
避障约束若选避障方案，不可用遥控器人为控制避障；只能用 Goal Tracking（以机器人与终点的相对位置为控制指令）或 Vel Tracking（用相对位置 PD 计算 Vel Command）。
03 / 比赛规则

赛制与流程

赛制3 次尝试，取最高分为最终分数。确定性评测：固定赛道 + 固定箱初始位 + 无域随机化 + 无观测噪声 + 固定摩擦。
任务流程① 起点感应线触发计时；② 机器人自主沿赛道向前推进，任意选择绕行 / 跨越的方式通过；③ 通过终点感应线或单次时长达 180 s（3 min）本局终止；④ 跌倒不终止——可自起恢复后继续。
避障约束避障方案不可遥控人为控制，只能 Goal Tracking 或 Vel Tracking。
提交提交物 = 推理代码 + 策略权重，命令行提交、自动评测上榜；提交次数不限，榜单取该队历史最高分。
04 / 评分标准（满分 100）

到达即得分，跨越拿奖励

到达终点即得主分；在此之上，成功跨越箱子并跳下、落地干净、箱沿降重心可叠加奖励；除双臂与双脚外的部位接触环境则扣分。各检测项独立、可叠加，总分 clamp 到 [0, 100]。

5.1 主成绩 — 到达终点

到达终点 = 60 分 （CoM 越过终点线 x ≥ +9 即得；绕行或跨越皆可。无速度 / 时间项）

5.2 跨越奖励（最高 +25，可叠加）

行为检测方式加分
成功跨越箱子CoM 在箱足迹内抬升至约箱高，再越过箱远沿+10
跳下时双脚同时落地双脚腾空后，在箱远侧同一步双脚同时触地+5
在箱沿有意识地降低重心跳下前在箱上、近远沿处 CoM 下蹲降高+10

三项独立检测、可叠加；跨越路线满拿 +25（总分上限 85）。绕行仅得到达分 60。

5.3 罚则 — 非臂非脚部位接触

接触部位扣分（t = 累计接触时长 / 秒）
躯干接触环境−5 × (1 + 0.2·t)
腿部接触环境−20 × (1 + t)

双臂 + 双脚豁免：手臂是合法操作部位、脚走路必然触地，二者接触不计罚。短暂接触即计基础罚（躯干 5 / 腿 20），持续拖地则随 t 增长。

05 / 提交方式

命令行提交，自动评测上榜

提交唯一通道是命令行 submit.py：交推理代码 + 策略权重，带队伍令牌。分数由主办方评测算出。提交次数不限，榜单取该队历史最高分。

准备工作：下载 submit.py（纯 Python 3 标准库，无需 pip install 任何依赖）。<网关仓库> 与 <队伍令牌> 由主办方公告 / 私发给各队；<队伍标识> 用你的队名（英文/数字/连字符）。

# 下载提交工具（单文件，零依赖） wget https://weil-limx.github.io/humanoid-board/submit.py
# 提交：推理代码 + 权重，带队伍令牌（若你的环境没有 python，请用 python3） python3 submit.py --competition humanoid --task ATEC-TaskF-OliEdu \ --repo <网关仓库> --team <队伍标识> --token <队伍令牌> \ --ckpt-file ./policy.pt --code-dir ./solution
# 查本队各次提交的状态与分数 python3 submit.py --status --repo <网关仓库> --team <队伍标识> --token <队伍令牌>

流程：① 实现推理代码 + 训练权重；② submit.py 打包上传、网关校验令牌； ③ 主办方评测自动跑一局并算分；④ 总分 + 明细 commit 到榜单 —— Dev 榜实时、Final 截止冻结。
